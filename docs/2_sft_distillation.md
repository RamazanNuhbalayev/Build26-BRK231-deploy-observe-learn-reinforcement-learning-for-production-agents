# Phase 2: SFT Distillation — Teaching Smaller Models

**Demo Time:** 10-12 minutes

---

## 🎯 What the Audience Should Walk Away With

1. How Supervised Fine-Tuning (SFT) distillation works
2. How to generate training data from a teacher model (GPT-5.4)
3. How to fine-tune GPT-4.1, GPT-4.1-mini, and GPT-4.1-nano via Azure AI Foundry
4. That student models can outperform the teacher through behavioral specialization
5. The simplicity: one API call to submit a fine-tuning job

---

## 🎬 Demo Script

### 1. The Concept — Knowledge Distillation (2 min)

**Key Message:** "We have a smart but expensive teacher (GPT-5.4). Let's train cheaper students to mimic its behavior on THIS specific task."

**Visual:**
```
Teacher (GPT-5.4, 64.5% on retail_quality)
    │
    │  Generates correct tool-call traces
    │  on 481 training scenarios
    ▼
┌─────────────────────────────────────────┐
│         Training Data (SFT)              │
│  481 scenarios with full conversations  │
│  Input: customer request                 │
│  Output: complete tool-call sequence     │
└─────────────────────────────────────────┘
    │              │              │
    ▼              ▼              ▼
GPT-4.1        GPT-4.1-mini   GPT-4.1-nano
(student)      (student)      (student)
```

**Why it works:**
- Teacher shows the *correct behavior* for each scenario
- Students learn the specific patterns (which tools, which order, which calculations)
- Students don't need general reasoning — just this task's behavior
- Result: specialized student > general teacher

---

### 2. Training Data Preparation (2 min)

**Show:** `data/retail_seed.jsonl` — 481 training scenarios

**Explain:**
- Each scenario: user message + GPT-5.4's full tool-call conversation
- Format: standard chat completion with tool calls embedded
- Generated by running GPT-5.4 against all training scenarios
- Validated: only scenarios where GPT-5.4 got the right answer

**Key Point:** "The training data IS the teacher's traces. Same format as production. No manual labeling needed."

---

### 3. Submit Fine-Tuning Job (3 min)

**Show:** `notebooks/phase3_submit_rft.ipynb` (SFT section)

**The API call is simple:**
```python
job = client.fine_tuning.jobs.create(
    model="gpt-4.1",
    training_file=train_file.id,
    validation_file=val_file.id,
    suffix="retail-sft",
    method={"type": "supervised_fine_tuning"}
)
```

**Key Points:**
- One API call to start training
- Azure manages compute, checkpointing, evaluation
- Training takes 1-2 hours
- Result: a new model deployment you can swap in

**Show:** The fine-tuned deployment names:
- `gpt-4-1-2025-04-14-ft-626f359e91b848f18189e24b3e8e39b9`
- `gpt-4.1-mini-2025-04-14.ft-2eb6d31ecccd4796bce8fd9ccdc1529d`
- `gpt-4-1-nano-2025-04-14-ft-ff0be97034bf49359c038f622e395d88`

### 4. Results — Students Beat the Teacher (3 min)

**Show:** Evaluation results comparison

| Model | Base | Fine-tuned | Improvement |
|-------|:---:|:---:|:---:|
| gpt-4.1 | 58.1% | N/A in this eval run | — |
| gpt-4.1-mini | 45.2% | **71.0%** | +25.8pp |
| gpt-4.1-nano | 1.6% | **37.1%** | +35.5pp |
| gpt-5.4 (teacher) | 64.5% | — | — |

**The Punchline:** "The students outperform the teacher! GPT-4.1-mini-finetuned reaches 71.0%, beating GPT-5.4 at 64.5%. Why? Because the student is specialized — it only needs to be good at THIS task."

**Cost Insight:** "And it's 3-5x cheaper per request. Smaller model, same (better!) quality."

### 5. Deploy and Test (2 min)

**Show:** Same agent code, just swap the model:
```bash
cd deploy
azd deploy retail-gpt-4-1-finetuned
```

**Demo:** Invoke the fine-tuned agent on a tricky scenario
```bash
azd ai agent invoke --agent-name retail-gpt-4-1-finetuned \
  --message "I want to exchange the yoga mat from ORD-007 for a different color"
```

**Compare:** Show how base gpt-4.1 might get this wrong (deny instead of exchange)
